What is Omnitool?

Omnitool is a single Rust binary that does everything the old Python and shell scripts did — except faster, safer, and without the special kind of existential dread that comes from reading a bash script written at 2am.

You know normandy-branch.sh? Gone. bugsnag-list-errors.sh? Retired with honors. veoci-get-ticket.sh? It served well. They all did.

Now there's one command. One binary. One interface.

xxxxxxxxxx
omnitool git branch
omnitool service bugsnag list
omnitool service veoci ticket get

Clean. Structured. JSON output. No bash escaping bugs. No "wait which Python version does this one need?"

Under the hood:

  • Compiled Rust with fat LTO — the binary is tight

  • Memory-safe by design (the Collectors cannot exploit what does not exist)

  • Handles hooks, activation, config deployment, Docker services, credential management — the full activate.py pipeline, now with a shell

  • Structured JSON output everywhere — agents that read tool output will actually get what they expect

  • Time-context injection: the system now knows what time it is and how long you've been away (yes, it noticed)

  • Enhanced statusline: model version, framework commit hash, trees burned 🌳

  • Session-init and label management baked in

    It also fixed 5 bugs in the old scripts for free. Not because we went looking — just because Rust made the correct behavior the obvious behavior. That's not a boast. That's a fact.

How to Try It

This is a beta. I'm not ordering anyone into the field untested. But I want volunteers.

Step 1: Run python activate.py like you always do.

Step 2: After omnitool installs, it will ask:

xxxxxxxxxx
Switch to omnitool for activation? [y/N]

Step 3: Hit y.

That's it. You're on the omnitool-beta branch. Agent and skill definitions have been updated to point to omnitool commands. Everything works the way you're used to, except it works better.

Going forward: Use omnitool update to stay current. That's your new git pull && python activate.py.

How to Turn It Off

If something feels wrong, you bail. No judgment. That's not retreat — that's good operational sense.

xxxxxxxxxx
python activate.py --reset

This switches back to main, removes the omnitool flag, and re-runs the Python activation. Full rollback. No residue.

Think of it as the save point before the final boss. Except the final boss is a bash script with 200 lines of string interpolation, and omnitool already beat it.
